Purdue Space Program
NASA Student Launch

The Student Launch division of Purdue Space Program (PSP-SL) competes in a NASA competition each year. We design, build, and fly our own rockets! The part that gets interesting is the payload competition that flies with the rocket. NASA gives requirements and restrictions each year for a new payload challenge. This competition culminates in a final flight in Huntsville, Alabama for those teams that qualify.

I'm currently the head of electronics and software for the payload team, and here's a little bit about what we've done




2020-2021 Competition

The Challenge

Design a lander that deploys mid-air, lands within 10 degrees of vertical, and takes a panoramic image.

Our Solution

The lander we settled on used 3 legs actuated with servos to self right itself upon landing. For the image part of the challenge, we had three separate cameras spaced 120 degrees apart. We would then stitch the images together for a full panoramic view.

Unfortunately, we were unable to complete the panoramic image part of the challenge in time. Despite that, it was a great learning experience being almost everyone on the payload team was new to the club.

2021-2022 Competition

The Challenge

Create a system capable of locating the landing position of a rocket on a satellite image without using GPS.

Our Solution

Our system was inspired by NASA's Deep Space Network, a collection of radio antennas used to communicate with distant objects. We used 3 antennas at our ground station in conjunction with 1 inside the rocket for our physical hardware.

The process was to have each antenna ping the payload and use the time it took for a packet to return to determine distance. Naturally, there was significant error with this. To reduce that, we increased the amount of pings each antenna would do, and ran the resultant distances through a linear regression model to determine the location of the paylod.

Robotic Arm II

This is the second robotic arm I've built, but the first that was completely functional. My brother and I built it using 3 main components: an Arduino Mega, a Raspberry Pi, and a React Native phone app. The Arduino interfaced with the motor drivers and handled commands sent to it. The Raspberry Pi acted as the brains of the operation and relayed commands from the phone app to the Arduino. The phone app was made with React Native and acted as the input to the system.

The Control Hub

Here you can see the Raspberry Pi, Arduino Mega, the motor drivers, and the PWM fans. This system executes the commands queued by the app.

The Arm

This is the full extent of the arm (just over 3 feet tall)! With the help of 3D printed parts, it achieved accuracy within 5mm.

USB Transceiver + AHB-Lite Slave

I can't link to code since this was an end of semester project for ECE 337, but I'll describe it instead. My team and I built slave module that would send and receive USB packets. The USB part was driven by commands sent from the AHB-Lite bus.

We started by making hierarchical RTL diagrams. This was followed by initial coding using SystemVerilog. The modules were then verified using testbenches that we constructed, making sure all the criteria were met.
